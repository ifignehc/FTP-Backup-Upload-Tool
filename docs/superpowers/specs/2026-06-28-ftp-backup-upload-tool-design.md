# FTP 备份上传工具设计规格

日期：2026-06-28

## 目标

做一个 Windows 免安装便携软件，用于公司服务器文件的备份、上传和检查。软件第一版面向 Windows，发布为可双击运行的 `.exe`，不需要安装服务。

第一版只支持 FTP。底层传输接口保留扩展点，后续可增加 SFTP / FTPS。

## 技术路线

采用 .NET 8 / WPF / 单文件便携 exe。

选择原因：

- WPF 适合做 Windows 原生三栏文件管理界面。
- 支持多选、右键菜单、复制、粘贴、删除、拖拽和进度反馈。
- 可以使用 Windows DPAPI 按当前用户加密保存 FTP 密码。
- 可以发布为便携单文件 `.exe`。

不采用 Electron / Tauri 作为第一版，因为三栏文件管理、FTP 操作、Windows 原生拖拽和便携打包会更重。不采用 Python / PySide 作为第一版，因为长期维护、复杂文件操作和打包稳定性不如 WPF。

## 核心概念

### 工序

一个工序是一套服务器和本地路径配置。用户可以创建多个工序，并在主界面切换。

每个工序包含：

- 工序名称
- 生产服务器 FTP 信息
- 起案服务器 FTP 信息
- 服务器根目录
- 本地根目录
- 默认路径清单文件
- 备份保存目录
- 备份文件夹命名模板
- 备份日志内容勾选项

### 路径规则

生产服务器根目录、起案服务器根目录、本地根目录下面的文件夹层级保持一致。

路径清单中的每一行按相对路径解释。例如：

```text
css/site.css
images/logo.png
templates/index.html
```

软件访问文件时，按当前工序自动拼接：

```text
生产服务器根目录 + 相对路径
起案服务器根目录 + 相对路径
本地根目录 + 相对路径
```

路径清单用于备份和 Check。上传按钮也按路径清单把本地文件映射到起案服务器对应位置。

## 主界面

主界面包含四个区域：

1. 顶部操作区
2. 路径清单区
3. 三个文件窗口
4. 日志窗口

线框图：`mockups/main-design-wireframe.html`

### 顶部操作区

包含：

- 工序选择
- 当前根目录提示
- 备份按钮
- 上传按钮
- Check 按钮
- 设置按钮

### 路径清单区

路径清单支持：

- 手动编辑
- 多行粘贴
- 导入 `.txt` 或 `.csv`
- 清空
- 保存清单

第一版每行只处理一个相对路径。空行和前后空格会被忽略。

路径清单中的反斜杠会规范化为斜杠，开头的斜杠会被移除。包含 `..` 的路径视为无效路径，必须在执行备份、上传或 Check 前提示错误，防止越过当前工序根目录。

### 文件窗口

主界面中间并排显示三个文件窗口：

- 生产服务器文件窗口
- 起案服务器文件窗口
- 本地文件窗口

生产服务器窗口：

- 可读
- 不可写
- 允许复制文件或文件夹
- 禁止粘贴、删除、上传和拖拽写入

起案服务器窗口：

- 可读可写
- 支持复制、粘贴、删除
- 支持从本地窗口拖拽或复制文件进入
- 不允许从本地上传到生产服务器

本地文件窗口：

- 可读可写
- 支持复制、粘贴、删除
- 支持多选和拖拽到起案服务器

### 日志窗口

日志窗口显示当前操作进度、结果和错误。日志等级包括：

- `Normal`
- `Warning`
- `Error`

日志需要显示时间、操作类型、相对路径、结果和错误说明。

## 备份功能

备份只从生产服务器下载文件，不修改生产服务器。

流程：

1. 读取当前路径清单。
2. 为本次备份创建一个本地备份文件夹。
3. 按路径清单逐条检查生产服务器对应文件。
4. 如果生产服务器存在文件，则下载到备份文件夹中，保留相对路径层级。
5. 如果生产服务器不存在文件，则认为该路径对应的是新文件，跳过备份。
6. 在主界面日志和备份日志中记录每条结果。

默认备份目录是桌面。默认备份文件夹命名模板：

```text
{yyyy}{MM}{dd}_{HH}{mm}{ss}_Backup
```

示例：

```text
20260628_153000_Backup
```

备份文件夹内容：

- 从生产服务器下载的文件
- 一份备份日志文件

备份日志可勾选写入字段：

- 相对路径
- 生产服务器完整路径
- 起案服务器完整路径
- 本地完整路径
- 文件大小
- 最后修改时间
- 操作结果
- 错误信息
- 备注

新文件的备注写为：`新文件，生产服务器不存在，跳过备份`。

## 上传功能

上传按钮只从本地根目录上传到起案服务器根目录，不涉及生产服务器。

流程：

1. 读取当前路径清单。
2. 按每条相对路径检查本地文件是否存在。
3. 检查起案服务器目标父文件夹是否存在。
4. 如果目标父文件夹不存在，弹窗报错并写日志。
5. 如果映射关系有效，则将本地文件上传到起案服务器对应相对路径。
6. 如果目标文件已存在，按第一版默认策略覆盖，并在日志中记录覆盖。

上传失败时，不继续静默吞掉错误。需要在主界面日志写明失败文件和原因。

### 手动上传

用户可以不用上传按钮，而是在文件窗口里手动操作：

- 本地窗口多选文件
- 复制到起案服务器窗口
- 拖拽到起案服务器窗口

手动上传同样必须遵守：

- 只能从本地上传到起案服务器
- 禁止向生产服务器写入
- 起案服务器目标路径必须在当前工序的服务器根目录下

## Check 功能

Check 检查路径清单、起案服务器和生产服务器之间的状态。

第一版按当前路径清单和起案服务器实际文件判断以下情况：

执行 Check 时，软件会递归扫描当前工序的起案服务器根目录，得到起案服务器实际文件集合；再把该集合与路径清单集合比较。路径清单中没有、但起案服务器根目录下存在的文件，判定为“路径缺失”。

### Normal：文件更新

条件：

- 路径清单中有该相对路径
- 起案服务器中存在对应文件

结果：

- 日志提示 `Normal`
- 说明起案服务器中上传了路径对应的文件

### Error：路径缺失

条件：

- 路径清单中没有该相对路径
- 起案服务器中存在新文件

结果：

- 日志提示 `Error`
- 说明起案服务器存在新文件，但路径清单缺失

### Warning：新路径旧文件

条件：

- 路径清单中有该相对路径
- 起案服务器中没有对应文件
- 生产服务器中有对应文件

结果：

- 日志提示 `Warning`
- 说明这是新路径，但生产服务器已有旧文件，可能没有上传到起案服务器

### Error：文件缺失

条件：

- 路径清单中有该相对路径
- 起案服务器中没有对应文件
- 生产服务器中也没有对应文件

结果：

- 日志提示 `Error`
- 说明路径对应文件缺失

## 设置界面

设置界面是主界面的二级窗口。

线框图：`mockups/settings-design-wireframe.html`

布局：

- 左侧：工序列表
- 右侧：当前工序配置
- 底部：保存 / 取消

左侧工序列表支持：

- 新增工序
- 复制工序
- 删除工序
- 重命名工序
- 搜索工序

右侧配置包含：

- 工序信息
- 生产服务器 FTP
- 起案服务器 FTP
- 根目录与备份
- 备份日志内容

生产服务器和起案服务器分别提供“测试连接”按钮。

## 配置与安全

配置优先保存在软件目录：

```text
config/appsettings.json
```

如果软件目录不可写，自动退到：

```text
%AppData%\FtpBackupUploadTool\appsettings.json
```

FTP 密码使用 Windows 当前用户加密保存。加密后的密码只能在同一台电脑、同一个 Windows 用户下自动解密使用。

换电脑后：

- 工序、服务器地址、账号、路径、备份设置可以保留
- 密码需要重新输入
