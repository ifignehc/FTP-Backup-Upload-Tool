# FTP 备份上传工具设计规格

日期：2026-06-28
更新：2026-06-30

## 目标

做一个 Windows 免安装便携软件，用于公司 FTP 文件的备份、上传和检查。软件面向 Windows，发布为可双击运行的 `.exe`，不需要安装服务。

当前版本只支持 FTP。底层传输通过 `IRemoteFileClient` 抽象封装，后续可以扩展 SFTP / FTPS，但当前 UI 和配置项均按 FTP 实现。

## 技术路线

采用 .NET 8 / WPF / Windows 便携 exe。

选择原因：
- WPF 适合 Windows 原生文件管理界面。
- 支持三栏文件窗口、路径输入、右键菜单、复制、粘贴、删除、拖拽和日志显示。
- 支持 Windows DPAPI 按当前用户加密保存 FTP 密码。
- 可通过脚本发布为 `win-x64` 便携 exe。

当前未采用 Electron / Tauri / Python GUI。第一版优先保证 Windows 文件操作、FTP 传输和便携打包稳定。

## 核心概念

### 工序

一个工序是一套生产服务器、起案服务器和本地操作环境的配置。主界面顶部可以选择已保存工序，切换后会重新加载对应 FTP 连接并刷新文件面板。

每个工序包含：
- 工序名称
- 生产服务器 FTP：IP / 域名、端口、账号、加密密码、服务器根目录、被动模式开关
- 起案服务器 FTP：IP / 域名、端口、账号、加密密码、服务器根目录、被动模式开关
- 备份保存目录
- 备份文件夹命名模板
- 备份日志字段勾选项
- 检查日志保存目录
- 检查日志命名模板

当前设置界面没有单独暴露“本地根目录”和“默认路径清单文件”。上传和 Check 使用主界面“本地文件”面板当前目录作为本地根目录。

### 路径规则

路径清单中的每一项都按相对路径解释。例如：

```text
css/site.css
images/logo.png
templates/index.html
```

软件访问文件时按当前工序自动拼接：

```text
生产服务器根目录 + 相对路径
起案服务器根目录 + 相对路径
本地文件面板当前目录 + 相对路径
```

路径清单解析规则：
- 支持多行输入。
- 也支持用英文逗号分隔。
- 会忽略空项和首尾空格。
- 反斜杠会规范化为斜杠。
- 开头的斜杠会移除。
- 重复路径按大小写不敏感去重。
- 不允许 Windows 绝对路径、UNC 路径、盘符路径。
- 不允许 `.` 或 `..` 路径段，防止越过当前根目录。

路径清单是备份、上传和 Check 的共同输入。主界面只提供可编辑文本框；当前版本没有导入、导出或保存路径清单按钮。

## 主界面

主界面包含四个区域：

1. 顶部操作区
2. 路径清单区
3. 三个文件窗口
4. 日志窗口

线框图参考：`mockups/main-design-wireframe.html`

### 顶部操作区

包含：
- 工序下拉选择
- 当前服务器根目录提示
- 备份按钮
- 上传按钮
- 检查按钮
- 设置按钮

软件启动时会读取配置。若存在已保存工序，则加载第一个工序并刷新三栏文件窗口；若没有工序，则在日志中提示先打开设置完成配置。

### 路径清单区

路径清单是一个多行文本框。用户可以手动输入或粘贴路径。

备份、上传、检查按钮只有在路径清单非空、且没有其他流程运行时才可执行。执行期间会禁止重复执行其他流程。

### 文件窗口

主界面中间并排显示三个文件窗口：

- 生产服务器
- 起案服务器
- 本地文件

每个文件窗口包含：
- 当前路径输入框
- 上一级按钮
- 进入按钮
- 文件列表
- 右键菜单
- 双击进入文件夹

文件列表显示名称、大小、修改时间。修改时间按北京时间显示。

生产服务器窗口：
- 只读。
- 支持复制文件。
- 禁止粘贴、删除、拖放写入。

起案服务器窗口：
- 可读写。
- 支持复制、粘贴、删除文件。
- 支持从本地窗口拖拽或复制文件进入。
- 支持从生产服务器复制文件到起案服务器。
- 写入时不会自动创建远端父文件夹；目标父文件夹不存在时记录错误。

本地文件窗口：
- 可读写。
- 默认打开当前 Windows 用户目录。
- 支持复制、粘贴、删除文件。
- 支持从生产服务器或起案服务器复制文件到本地。
- 作为上传和 Check 的本地根目录来源。

当前版本只处理普通文件的复制、上传和删除；右键菜单会过滤文件夹。

### 日志窗口

日志窗口显示当前操作进度、结果和错误。日志格式包含时间、等级、操作名、路径和消息。

日志等级包括：
- `Normal`
- `Warning`
- `Error`

日志窗口中的常见操作名：
- `Backup`
- `Upload`
- `Check`
- `Copy`
- `Delete`

## 备份功能

备份只从生产服务器下载文件，不修改生产服务器。

流程：
1. 读取当前路径清单。
2. 根据备份保存目录和命名模板创建一个本地备份文件夹。
3. 按路径清单逐条检查生产服务器文件。
4. 如果生产服务器存在文件，则下载到备份文件夹，保留相对路径层级。
5. 如果生产服务器不存在文件，则认为是新文件，跳过备份。
6. 在主界面日志中记录每条结果。
7. 在备份文件夹内写入一份 Markdown 备份日志。

默认备份保存目录：

```text
%USERPROFILE%\Desktop
```

默认备份文件夹命名模板：

```text
{yyyy}{MM}{dd}_{HH}{mm}{ss}_Backup
```

示例：

```text
20260628_153000_Backup
```

模板渲染结果必须是单个安全文件夹名，不能是绝对路径，不能包含路径分隔符。

备份日志文件名与备份文件夹名一致，扩展名为 `.md`。日志支持按设置勾选字段：
- 相对路径
- 生产完整路径
- 起案完整路径
- 本地完整路径
- 文件大小
- 最后修改时间
- 操作结果
- 错误信息
- 备注

生产服务器不存在的文件在备份日志中标记为 `Skipped`，备注为“新文件，生产服务器不存在，跳过备份”。

## 上传功能

上传按钮只从本地文件面板当前目录上传到起案服务器根目录，不涉及生产服务器。

流程：
1. 读取当前路径清单。
2. 将每条相对路径映射到“本地文件”面板当前目录下的文件。
3. 如果本地文件不存在，记录 `Error`，继续处理后续路径。
4. 如果起案服务器目标父文件夹不存在，记录 `Error`，不自动创建远端文件夹。
5. 如果目标文件已存在，则覆盖上传，并记录“覆盖上传完成”。
6. 如果目标文件不存在，则上传并记录“上传完成”。

上传使用临时远端文件再替换目标文件，尽量避免上传中断造成目标文件被截断。

## 手动文件操作

用户也可以通过三个文件窗口手动复制、粘贴、拖拽和删除。

规则：
- 生产服务器面板只读，不能作为写入目标。
- 起案服务器可作为写入目标。
- 本地文件窗口可作为写入目标。
- 同一面板内复制会跳过。
- 复制到起案服务器时，目标父文件夹必须已存在。
- 复制到本地时，如果本地目标父文件夹不存在，会自动创建。
- 删除只支持起案服务器文件和本地文件，不支持生产服务器。

手动从生产服务器复制到起案服务器时，会先下载到本机临时文件，再上传到起案服务器。

## Check 功能

Check 用于检查路径清单、起案服务器、生产服务器和本地文件之间的状态。

执行 Check 时会先按路径清单去重，然后逐条判断：

### Normal：文件更新

条件：
- 路径清单中有该相对路径。
- 起案服务器中存在对应文件。

结果：
- 记录 `Normal`。
- 消息说明起案服务器已上传对应文件，并显示起案服务器文件修改日期。

### Warning：新路径旧文件

条件：
- 路径清单中有该相对路径。
- 起案服务器中没有对应文件。
- 生产服务器中有对应文件。

结果：
- 记录 `Warning`。
- 消息说明本次没有更新到起案服务器，但生产服务器已有旧文件。

### Error：文件缺失

条件：
- 路径清单中有该相对路径。
- 起案服务器中没有对应文件。
- 生产服务器中也没有对应文件。

结果：
- 记录 `Error`。
- 消息说明起案服务器和生产服务器均没有对应文件。

### Error：路径清单漏写

条件：
- 本地文件面板当前目录下存在某个文件。
- 该文件的相对路径没有写入路径清单。

结果：
- 记录 `Error`。
- 消息说明“本地文件没有写入路径清单”。

Check 完成后会写入 Markdown 检查日志。日志内容包含：
- CheckTime
- UpdatedFiles 数量
- Warnings 数量
- Errors 数量
- Updated Files 明细
- Warnings 明细
- Errors 明细

默认检查日志保存目录：

```text
%USERPROFILE%\Desktop
```

默认检查日志命名模板：

```text
{yyyy}{MM}{dd}_{HH}{mm}{ss}_Check
```

检查日志会自动补 `.md` 扩展名。如果同名文件已存在，会追加 `-2`、`-3` 等后缀避免覆盖。

## 设置界面

设置界面是主界面的二级窗口。

线框图参考：`mockups/settings-design-wireframe.html`

布局：
- 左侧：工序列表。
- 右侧：当前工序配置。
- 底部：取消 / 保存设置。

左侧工序列表支持：
- 新增工序
- 复制工序
- 删除工序

当前版本没有单独的搜索、重命名按钮。修改工序名称后保存，即视为更新该工序名称。

右侧配置包含：
- 工序基础信息
- 生产服务器 FTP
- 起案服务器 FTP
- 服务器根目录、备份与检查日志
- 备份日志内容

生产服务器和起案服务器配置项：
- IP / 域名
- 端口
- 账号
- 密码
- 是否记住密码
- 被动模式

当前版本没有“测试连接”按钮。保存设置后，主界面会加载配置并刷新文件面板；连接或列表失败会写入主界面日志。

## 配置与安全

配置优先保存在软件目录：

```text
config/appsettings.json
```

如果软件目录不可写，自动退回到：

```text
%AppData%\FtpBackupUploadTool\appsettings.json
```

配置保存采用临时文件写入后替换，降低写入中断导致配置损坏的风险。

FTP 密码使用 Windows 当前用户 DPAPI 加密保存。加密后的密码只能在同一台电脑、同一个 Windows 用户下自动解密使用。

换电脑或换 Windows 用户后：
- 工序、服务器地址、账号、路径、备份设置可以保留。
- 已保存密码可能无法解密，需要重新输入。

如果不勾选“记住”密码，保存配置时不保留该服务器密码。
